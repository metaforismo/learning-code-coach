# Contributing

Contributions should keep the skill practical, concise, and safe for real development work.

## Principles

- Prefer short coaching patterns over long tutorials.
- Keep explanations tied to concrete work.
- Support many technical domains, with coding as the main use case.
- Avoid shaming language around AI use.
- Do not add telemetry, personal-data collection, or hidden persistence.
- Validate every change with `python3 scripts/validate_skill.py`.

## Good Changes

- Better calibration questions.
- More precise coaching patterns for a stack or workflow.
- Clearer anti-dependency guidance.
- Examples that teach how to verify AI output.

## Changes To Avoid

- Framework-specific bloat in `SKILL.md`.
- Generic programming tutorials.
- Advice that encourages unverified copy-paste.
- Silent storage of learner profiles or personal learning data.
